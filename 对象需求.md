### 节点信息表

```
节点属性:
	"nodeID": int		# 节点ID		  # PK
    "nodeName": str		# 节点名称		 # unqiue
    "ipAddress": str	# IP地址
    "port": str			# 端口号
    "status": str		# 节点运行状态	
    "logo": str			# 图片转base64

方法:
	1. 返回所有节点对象组成的列表
	2. 给定节点对象，添加到数据库
	3. 给定节点对象，根据其名字修改数据库里对应项
	4. 给定节点名字删除数据库里对应项目
```



### 用户信息表

```
用户属性:
	"userID": int		# 用户ID		  # PK
    "userName": str		# 用户姓名
    "userAccount": str	# 用户账户		# unqiue
    "password": str		# 用户密码
    "userType": str		# 用户类型		
    "institution": str	# 机构名		 # FK

方法:
	1. 返回所有用户对象组成的列表
	2. 给定用户类型，查询所有满足的用户对象组成的列表
	3. 给定用户账户，查询所有满足的用户对象组成的列表
	4. 给定用户对象，添加到数据库
	5. 给定用户对象，根据其账户修改数据库里对应项
	6. 给定用户账户删除数据库里对应项目
```





### 场景

```
场景信息表:
    属性:
        "sceneID": int		    # 场景ID		  # PK
        "sceneName": str		# 场景名称
        "institution": str		# 机构名 		# FK
        "target": str			# 目标字段

目标字段取值释义信息表:
	属性:
		"ID": int		    	# ID		  # PK
        "sceneID": int		    # 场景ID		  # FK
        "value": str			# 取值
        "label": str			# 取值说明

注意两个表结合成视图使得能够查到每个场景对象如下:
    scene:{
        "sceneName": str	# 场景名称
        "target": str		# 目标字段
        # 目标字段取值释义
        "describe": [{
            "value": str	# 取值
            "label": str	# 取值说明
            }		# 值1及说明
            {...}	# 值2及说明
            ...
        ]
    }
方法:
	1. 返回所有场景对象组成的列表
	2. 给定机构，查询该机构对应的场景对象组成的列表
	3. 给定场景对象，添加到数据库
	4. 给定场景对象，根据其账户修改数据库里对应项
	5. 给定场景名称删除数据库里对应项目
```





### 数据字典

```
数据字典信息表:
	"dictID": int			# 数据字典ID		  # PK
	"institution": str		# 机构名 		# FK
	
	
数据字典属性表:
	"attrID": int		# ID		  # PK
	"dictID": int		# 数据字典ID		  # FK
    "attrName": str		# 属性名称
    "cnName": str		# 中文名
    "describe": str		# 释义
    "dataType": str		# 数据类型
    "class": str		# 类别
    "provider": str		# 提供方
    "area": str			# 取值范围


方法:
	1. 返回所有数据字典对象组成的列表
	2. 给定机构，查询该机构对应的数据字典对象
	3. 给定机构，查询该机构对应的数据字典对象是否存在
	4. 给定机构和数据字典对象，添加到数据库
	5. 给定机构和数据字典对象，修改数据库里对应项
```





### 数据集表

```
数据集信息表:
	"datasetID": int	# 数据集ID		  	# PK
	"datasetName": str	# 数据集名称
	"userID": int		# 用户ID		  	# FK
	"sceneID": int		# 场景ID			# FK
	"alignedNum": int	# 对齐样本量
	"timestamp": str	# 建立时间
	
数据集属性表:	
	"datasetID": int	# 数据集ID		  	# FK
    "attrID": int		# attrID		  # FK

数据集提供方表:	
	"datasetID": int	# 数据集ID		  	# FK
    "nodeID": int		# nodeID		  # FK
    
注意需要多表结合成视图使得能够查到每个数据集对象如下:    
dataset:{
	"userAccount": str	# 用户账户
	"sceneName": str	# 场景名称
    "datasetName": str	# 数据集名称
    "alignedNum": int	# 对齐样本量
    # 数据集包含的属性
    "dict":[{
            "provider": str		# 数据提供者名称
            "attributes": [str, str, ...]	# 采用的属性
         }	 # 数据提供1
        {...}# 数据提供2
        ...
    ]
    "timestamp": str	# 建立时间 "YY.MM.DD HH:MM"
} # 数据集
        
        
方法:
	1. 给定用户账户和场景名，查询所有满足的数据集合对象组成的列表
	2. 给定数据集名称，删除数据库对应信息
	3. 给定数据集对象，添加到数据库类
```



### 任务表

```
任务信息表:
	"taskID": int		# 任务ID		  	# PK
	"datasetID": int	# 数据集ID		  	# FK
	"userID": int		# 用户ID		  	# FK
	"sceneID": int		# 场景ID			# FK
	"timestamp": str	# 建立时间

任务训练表:
	"taskID": int		# 任务ID		  	# FK
	"status": str		# 训练状态
	"accuracy": float
    "F1": float
    "precision": float
    "recall": float
    "FBeta": float
    "logLoss": float
    "Roc": float
        
方法:
	1. 给定用户账户，场景名，查询所有满足的任务对集合对象组成的列表
	2. 给定任务名称，删除数据库对应信息
	3. 给定用户账户、场景名和数据集名，添加任务对象到数据库类
	4. 给定任务名称，添加任务训练对象到数据库类
```
